Имеется список пользователей; каждому из них соответствует список email-ов, например:
user1 -> xxx@ya.ru, foo@gmail.com, lol@mail.ru
user2 -> foo@gmail.com, ups@pisem.net
user3 -> xyz@pisem.net, vasya@pupkin.com
user4 -> ups@pisem.net, aaa@bbb.ru
user5 -> xyz@pisem.net
 
Количество пользователей и количество email-ов у каждого из них никак не ограничено.
 
Считается, что если у двух пользователей есть хотя бы один общий email, то это один и тот же пользователь. 
Требуется придумать и реализовать алгоритм, выполняющий слияние пользователей; на выходе должен получиться 
список «уникальных» пользователей с их email-ами. В качестве имени «объединенного» пользователя можно брать любое из исходных имен. 
Список email-ов каждого «объединенного» пользователя должен содержать все его уникальные email-ы.
 
Возможный ответ на пример выше:
user1 -> xxx@ya.ru, foo@gmail.com, lol@mail.ru, ups@pisem.net, aaa@bbb.ru
user3 -> xyz@pisem.net, vasya@pupkin.com
 
Требования к решению:
1. Асимптотическая сложность по времени должна быть линейной или близкой к линейной; оценку этого времени нужно приложить к 
описанию решения.
2. Задачу необходимо реализовать на языке Java - достаточно как консольное приложение, но не возбраняется использовать 
Spring и/или любые другие библиотеки из экосистемы Java.
3. Входные данные должны читаться из stdin в каком-нибудь разумном формате, и выводиться в stdout в таком же формате (см. пример).
4. Приложение должно собираться с помощью Maven или Gradle.
